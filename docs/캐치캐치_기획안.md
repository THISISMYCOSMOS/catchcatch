# 캐치캐치 (CatchCatch)

> 결제 직전, "지금 이 가격이 정말 살 만한가?"에 답하는 **진짜 할인 검증 서비스**

## 1. 프로젝트 소개

기존 가격 비교 서비스는 "어디가 가장 싼가"까지만 답합니다. 캐치캐치는 한 걸음 더 나아가 **화면에 표시된 세일이 진짜 할인인지**를 검증합니다. 상품 링크 하나를 붙여넣으면 쿠폰·배송비·용량·구성품·직전 세일가까지 반영한 **실질 할인 정도**와 **구매 판단 결론**을 돌려줍니다.

**핵심 사용자**: 화장품을 정기적으로 구매하고 올영세일·쿠팡 할인 시기를 챙기지만, 매번 여러 플랫폼을 손으로 비교하기는 귀찮은 사용자

### 핵심 가치

1. **진짜 할인 검증** — 표시 할인율이 아닌, 쿠폰·배송비·용량·구성품·직전 세일가를 반영한 실질 할인 계산
2. **최저가와 구매 추천의 분리** — 표시상 최저가와 구매 안정성이 확인된 판매처를 나눠 제시
3. **판단 근거 제시** — 결론만 던지지 않고 왜 그런 결론인지 근거와 신뢰도를 함께 표시

## 2. 문제 정의

- 화장품은 세일이 1년 내내 반복되어 "50% 할인" 같은 숫자가 실제 이득을 보장하지 않음
- 같은 제품도 올리브영·쿠팡·무신사 뷰티·공식몰에 **다른 가격·용량·구성**으로 흩어져 있음
- 쿠폰 적용가, 배송비, 단품 vs 기획세트, 증정품, 판매자 신뢰도까지 같은 기준으로 환산해야 정확한 비교가 가능하지만, 사람이 손으로 하기엔 번거롭고 실수하기 쉬움
- 결과적으로 사용자는 결제 직전에 탭을 여러 개 열고 가격을 옮겨 적는 피로한 과정을 반복

**캐치캐치는 이 과정을 링크 하나로 줄입니다.**

## 3. 사용자 흐름

```
회원가입 → 판단 기준 3가지 선택 → 상품 링크 입력
→ 판매처 4곳 비교 분석 → 최종 결론 + 가격 차트 + 기준별 결과 확인
→ (필요 시) 유사 상품 추천 확인 → 관심 상품 저장/알림 → 외부 판매처 이동
```

- 회원가입 시 6가지 판단 기준 중 **3가지 선택**: 가장 적게 결제하기 / 지금 사기 좋은지 확인하기 / 본품·추가 용량 많이 받기 / 용량 대비 오래 쓰기 / 한정 사은품 챙기기 / 필요한 만큼만 사기
- 선택한 기준은 최종 결론에 더 큰 비중으로 반영되며, 기준별 결과도 별도 제공
- 모바일에서는 쇼핑 앱의 공유 버튼으로 링크를 바로 넘겨 검증 가능

## 4. 핵심 기능

| 기능 | 설명 |
|---|---|
| 상품 정보 자동 추출 | 링크에서 상품명·브랜드·용량·옵션·구성품 정리 |
| 판매처 비교 | 올리브영·무신사 뷰티·쿠팡·브랜드 공식몰의 동일 상품 매칭 및 비교 |
| 실구매가 계산 | 공개 쿠폰·배송비 반영, 판매처별 실제 결제 금액 산출 |
| 용량당 가격 환산 | 모든 가격을 1ml/1g 단위로 통일해 비교 |
| 기획세트·증정품 판단 | 화장품 구성(본품·리필·미니어처)은 용량 계산에 포함, 비화장품 사은품(키캡·파우치 등)은 별도 혜택으로 표시 |
| 진짜 할인율 계산 | 최근 평균가·직전 세일가 대비 실질 할인 정도 산출 |
| 최종 결론 제공 | **저점매수 / 원가에 가까움 / 적당히 살 만함** 중 하나 + 판단 근거 |
| 유사 상품 추천 | 진짜 할인이 아니거나 구매 가치가 낮으면 대안 제시 |
| 관심 상품 저장·알림 | 가격 변동·쿠폰 변경·세일 시작 알림 |

## 5. 신뢰 장치

- **판단 신뢰도 표시**: 가격 정보의 충분성, 구성 확인 여부에 따라 신뢰도를 함께 제시 — 애매하면 애매하다고 보여줌
- **동일 상품 매칭 확신도**: 확신도가 높으면 비교 포함, 낮으면 추천 제외 또는 주의 문구 표시
- **할인 함정 알림**: 표시 할인율은 크지만 실제 이득이 작은 경우 경고
- **브랜드 가격 투명도 뱃지**: 가격·구성 정보의 명확성만 평가 (제품 품질·브랜드 평가 아님, 광고와 무관)
- **원칙**: 확인할 수 없는 정보는 단정하지 않고 "추정값" 또는 "확인 어려움"으로 표시

## 6. 결과 화면 구성

1. **최종 결론** (최상단) — 저점매수 / 원가에 가까움 / 적당히 살 만함 + 근거
2. **가격 변화 차트** — 현재 가격이 최근 흐름·직전 세일 대비 어느 위치인지
3. **기준별 결과** — 사용자가 선택한 3가지 기준 각각의 판단
4. **판매처별 비교표** — 표시가·쿠폰 적용가·배송비 포함가·용량당 가격·매칭 확신도
5. **직전 세일 비교 / 유사 상품 추천 / 저장·알림**

## 7. 기술 스택

| 영역 | 기술 |
|---|---|
| 프론트엔드 | Next.js, TypeScript, Tailwind CSS, shadcn/ui |
| 백엔드 | FastAPI (Python) |
| 데이터베이스 | Supabase PostgreSQL |
| AI | OpenAI API |
| 검색 연동 | 네이버 쇼핑 검색 API 등 (동일 상품 후보 탐색용) |
| 차트 | Recharts |
| 배포 | Vercel, Render, Supabase |

### 주요 API

- `POST /analyze-link` — 링크 분석
- `GET /analysis/{id}` — 결과 조회
- `GET /products/{id}/offers` — 판매처별 오퍼 조회
- `POST /user-criteria` — 판단 기준 저장
- `POST·GET /saved-products` — 관심 상품 저장·조회
- `POST /alerts` — 알림 조건 저장

### 데이터베이스 설계

`products`(상품), `sellers`(판매처), `product_offers`(판매처별 가격·쿠폰·배송비·구성), `price_snapshots`(가격 히스토리), `product_matches`(매칭 확신도), `analysis_results`(최종 결론·기준별 결과·신뢰도), `user_criteria`(선택 기준), `saved_products`(관심 상품), `alert_rules`(알림 조건)

## 8. AI와 계산 로직의 역할 분담

- **백엔드 계산 로직**: 실구매가·용량당 가격·진짜 할인율처럼 공식이 명확한 값 — AI가 임의로 만들지 않음
- **AI**: 상품 페이지 해석, 단품·기획세트·증정품 구분, 동일 상품 매칭 판단, 계산 결과와 선택 기준을 종합한 최종 결론·근거 생성
- 데이터가 부족하면 단정하지 않고 신뢰도를 낮추며, 제품 품질·성분·효능은 판단하지 않음

## 9. 데모 범위

**구현하는 것**
- 회원가입 및 판단 기준 3가지 선택
- 상품 링크 입력 → 샘플 데이터 조회 → 실구매가·할인율 계산 → 결과 화면 반환
- AI를 이용한 단품·기획세트·증정품 구분
- AI를 이용한 최종 결론·근거 생성
- 판매처별 비교표, 가격 차트, 기준별 결과 표시

**제외 또는 샘플로 대체하는 것**
- 전체 판매처 실시간 가격 수집·자동 갱신
- AI를 이용한 전체 상품 실시간 검색·매칭
- 완전 자동화된 가격 히스토리 축적, 운영 수준 푸시 알림

가격·판매처 조건은 대표 상품 기준의 **고정 샘플 데이터**로 구성하며, "모든 것을 AI가 실시간으로 한다"고 과장하지 않습니다.

## 10. 차별점

| 비교군 | 강점 | 캐치캐치와의 차이 |
|---|---|---|
| 올리브영·쿠팡 | 자체 쿠폰·멤버십 혜택 | 타 플랫폼·직전 세일 조건까지 같은 기준으로 검증하지 않음 |
| 네이버쇼핑·다나와·에누리 | 여러 몰 최저가 탐색 | 용량·구성품·직전 세일가·매칭 확신도를 묶은 구매 판단은 없음 |
| 화해·글로우픽 | 성분·리뷰 기반 제품 선택 | "지금 이 가격이 진짜 할인인지" 검증에는 초점 없음 |

캐치캐치는 이 서비스들을 대체하지 않고, 그 사이에 비어 있던 **"구매 직전 가격 판단"** 영역을 맡습니다.

## 11. 수익모델

1. **제휴 링크 기반 성과형 수익** — 결과 화면에서 외부 판매처로 이동해 구매가 이어지면 수익 발생 (직접 결제·주문은 처리하지 않음)
2. **유사 상품 추천 영역의 광고·스폰서 노출** — 추천 영역에 한정

**중립성 원칙**: 광고는 할인 판단·최종 결론·비교 순위·최저가 표시에 영향을 주지 않으며, 광고 상품은 일반 추천과 명확히 구분 표시

## 12. 향후 계획

- **데이터 확보**: 공식 API·제휴 데이터·검색 API로 확장하되, 확인된 값과 제한적 값을 구분 표시
- **가격 히스토리**: 스냅샷 축적으로 최근 평균가·직전 세일가·세일 흐름 계산
- **매칭 정확도**: 리뉴얼 전후·색상 번호·표기 차이까지 비교, 사용자 피드백으로 보정
- **AI 판단 확대**: 데이터가 쌓일수록 AI 담당 범위를 넓히되 "확인된 근거 안에서만 판단" 원칙 유지
- **카테고리 확장**: 화장품 안정화 후 인접 카테고리로 확장 — 범위보다 검증 가능성 우선
